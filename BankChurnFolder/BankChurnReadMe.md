## BankChurnReadMe

## Code Overview
- This project analyzes bank customer churn data using Python in a Jupyter Notebook, focusing on identifying patterns and predicting customer attrition.

-The workflow includes data loading, cleaning, validation, exploratory data analysis, and statistical testing, followed by building machine learning models to predict churn.

-Data cleaning steps include removing invalid values (e.g., unrealistic ages, credit limit inconsistencies), handling missing data, and standardizing categorical features to ensure accuracy in analysis.

-Feature exploration includes analyzing variables such as customer inactivity, credit utilization, and tenure to understand their relationship with churn.

-The project uses libraries such as pandas, NumPy, scikit-learn, and scipy for data processing and modeling, along with seaborn and matplotlib for visualization.
## Results


-Identified key drivers of churn, including customer inactivity, credit utilization patterns, and engagement metrics.
Built predictive models to classify churned vs. retained customers, achieving strong confidence in model performance.
Discovered trends showing that lower engagement and higher inactivity periods correlate strongly with increased churn risk.

-Visualized insights through correlation plots and statistical tests, making patterns easy to interpret and actionable.
Findings can be used to support targeted retention strategies and improve customer lifecycle management.

## Notes
-All analysis and modeling are contained within the Jupyter Notebook file.

-Code is structured step-by-step to reflect a real-world data analysis workflow, from raw data to final insights.
